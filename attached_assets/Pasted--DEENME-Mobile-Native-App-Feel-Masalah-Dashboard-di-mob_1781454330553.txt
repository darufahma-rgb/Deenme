# DEENME — Mobile: Native App Feel

## Masalah
Dashboard di mobile terlihat seperti website yang diperkecil, bukan app native. Root issues:
1. `.h1` font 28px → tanggal wrap 3 baris memakan ruang
2. Prayer grid 3 kolom terlalu sempit → tombol gepeng, teks terpotong
3. Header "next prayer" card duduk di sebelah tanggal → berantakan
4. Timbangan Amal tidak terbaca dengan baik
5. Padding content terlalu besar untuk layar kecil

---

## Perubahan di `src/dashboard.jsx`

### Update struktur header DashboardPage

Cari blok JSX header (sekitar baris 940–974):

```jsx
{/* Header */}
<div style={{ display: 'flex', alignItems: 'flex-end', justifyContent: 'space-between', gap: 20, marginBottom: 18 }}>
```

**Ganti** `style` div pembungkus itu dengan className:

```jsx
{/* Header */}
<div className="dashboard-header-row" style={{ display: 'flex', alignItems: 'flex-end', justifyContent: 'space-between', gap: 12, marginBottom: 18 }}>
```

Dan pada card "next prayer", tambahkan className:

```jsx
<div className="card next-prayer-card" style={{ padding: '11px 16px', display: 'flex', alignItems: 'center', gap: 10, flexShrink: 0 }}>
```

## Hasil yang Diharapkan

| Elemen | Sebelum | Sesudah |
|--------|---------|---------|
| Tanggal `.h1` | 28px → wrap 3 baris | 22px → 1–2 baris bersih |
| "Next prayer" card | Melayang di kanan tanggal, berantakan | Full width di bawah tanggal |
| Prayer grid | 3 kolom sempit | 2 kolom lebih luas |
| Tombol Tepat/Telat/Qadha | Gepeng 30px | Min 36px, terbaca |
| Padding content | 16px | 14px + no horizontal overflow |
| Bottom nav | 64px | 58px lebih compact |
| Input font | Apapun | 16px (prevents iOS auto-zoom) |
| Scroll | Scrollbar visible | Hidden, smooth native feel |
