BUG 2 — Auto-timestamp saat sholat dicatat
When user taps "Tepat", "Telat", or "Qadha" on a prayer, automatically fill the time input with the current WIB time at that exact moment.
In setStatus function inside src/App.jsx, update to auto-fill time:
jsconst setStatus = (k, val) => {
  setPrayers((p) => {
    const n = { ...p };
    if (val) n[k] = val;
    else delete n[k];
    return n;
  });

  if (val) {
    // Auto-fill current WIB time when prayer is marked
    const now = new Date();
    const wibTime = now.toLocaleTimeString('id-ID', {
      timeZone: 'Asia/Jakarta',
      hour: '2-digit',
      minute: '2-digit',
      hour12: false
    });
    setTimes((tm) => ({ ...tm, [k]: wibTime }));
  } else {
    // If unchecked, clear the time
    setTimes((tm) => {
      const n = { ...tm };
      delete n[k];
      return n;
    });
  }
};
This means:

Tap "Tepat" at 04:38 → time input auto-fills "04:38"
Tap "Telat" at 07:15 → time input auto-fills "07:15"
Tap "Qadha" at 13:00 → time input auto-fills "13:00"
User can still manually edit the time after auto-fill
Untapping clears the time


Do not change any other functionality. Only fix these two bugs.
